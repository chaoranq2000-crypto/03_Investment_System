# Operating Evidence Qualification Policy

## 状态

| 状态 | 含义 |
|---|---|
| `confirmed` | 有数值、单位、期间、证据定位、来源等级和足够置信度 |
| `bounded_estimate` | 有上下界、方法、单位、期间、证据或可审计推导 |
| `missing` | 未获得足够输入；必须生成问题与 backflow |
| `conflicting` | 存在不可消解冲突；不得进入经营模型 |

`missing` 和 `conflicting` 永远不能通过更长文本、更多章节或更高的其他分数抵消。

## 独立业务暴露

一个业务线只有在以下条件同时满足时，才可用于 SOTP 或同业经营比较：

1. 有独立量化指标；
2. 有 evidence IDs；
3. 业务边界可与宽口径分部协调；
4. 重叠已经分配或扣除；
5. 期间和单位明确。

产品存在、技术认证或管理层叙事本身不等于独立收入或利润暴露。

## 估值方法资格

- **Peer**：至少三家定义、期间和指标口径兼容的同业；
- **DCF**：经营覆盖通过，至少三期经营现金流和资本开支，营运资金桥及折现输入完整；
- **SOTP**：每个重大业务线均有独立财务、方法和已解决重叠。

资格失败时仍可保留反向估值和情景估值，但不得伪造 Peer、DCF 或 SOTP 精度。
