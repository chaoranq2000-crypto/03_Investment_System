# Stock Report Target Standard — 样例级个股报告目标标准

## 1. 报告定位

样例级报告不是“把数据堆长”，而是要形成可追溯、可复核、可更新的研究判断链。

每篇报告必须回答：

```text
1. 这家公司最重要的研究主线是什么？
2. 财务表现是表观变化还是真实经营变化？
3. 哪些业务真正贡献收入、毛利和弹性？
4. 公司所处细分行业的关键变量是什么？
5. 未来三年盈利预测由哪些变量驱动？
6. 当前估值是便宜、合理、偏贵还是已透支？
7. 技术面、情绪面和事件面在什么阶段？
8. 什么证据会证明或证伪本文判断？
```

## 2. 结构标准

```text
标题：价值发现 / 个股深度：股票代码 公司简称

前言
一、财务概览
二、业务拆分
三、行业分析
四、盈利预测
五、估值分析
六、技术分析
七、情绪分析
八、事件驱动
九、研究结论、风险与跟踪清单
附录：Evidence Map / Open Questions
```

## 3. 前言标准

前言不是公司简介。前言必须形成一句研究主线。

合格结构：

```text
公司身份定位
+ 当前关键变化
+ 市场分歧
+ 本文核心判断
+ 最大不确定性
```

示例句式：

```text
{{company_name}} 是一家 {{历史身份/业务标签}}，当前正在经历 {{关键变化}}。
市场分歧集中在 {{分歧点1}} 和 {{分歧点2}}。
本文认为，真正需要跟踪的不是 {{表面变量}}，而是 {{核心变量}}。
```

## 4. 财务概览标准

必须包括：

```text
- 最近 3 年 + 最近一期收入、归母净利、扣非净利、毛利率、净利率。
- 现金流质量：经营现金流、回款、应收、存货、资本开支。
- 资产负债表：现金、债务、负债率、流动性。
- ROE / ROIC / 周转率等关键指标。
- 表观利润和真实经营利润差异。
```

必须形成判断，而不是只列数字：

```text
- 增长来自量、价、结构、并表、周期还是一次性收益？
- 利润质量是否被非经常性损益扭曲？
- 现金流是否匹配利润？
- 资产负债表是否支持下一轮扩张？
```

## 5. 业务拆分标准

每个业务段至少包括：

```text
业务名称
收入 / 占比 / 增速
毛利率 / 毛利贡献
产品或服务
客户或下游
产能 / 订单 / 项目
竞争地位
未来 1-3 年关键变量
风险和反证
```

表格模板：

```markdown
| 业务 | 收入 | 占比 | 毛利率 | 毛利贡献 | 增长驱动 | 关键证据 | 置信度 |
|---|---:|---:|---:|---:|---|---|---|
```

禁止事项：

```text
- 未披露分业务收入时，不得编造占比。
- 公司整体收入不能直接归因到某个细分。
- 管理层表示“积极布局”不能等同于已贡献收入。
```

## 6. 行业分析标准

行业部分可先做轻量版，但必须围绕公司业务相关细分。

最低结构：

```text
3.1 细分定义与需求驱动
3.2 供给格局和竞争壁垒
3.3 价格、利润池和价值链位置
3.4 公司在行业中的位置
3.5 关键验证指标
```

每个细分至少回答：

```text
- 行业为什么现在重要？
- 景气度由什么驱动？
- 公司赚的是行业哪一段的钱？
- 竞争对手是谁？
- 未来 3-6 个月跟踪什么指标？
```

## 7. 盈利预测标准

必须有明确模型，而不是口头预测。

最低字段：

```yaml
forecast_model:
  period: [2026E, 2027E, 2028E]
  revenue:
  gross_margin:
  operating_profit:
  net_profit_attributable:
  eps:
  key_assumptions:
  sensitivity:
  evidence_basis:
  confidence:
```

正文必须解释：

```text
- 三年预测的核心变量。
- 与一致预期或历史趋势的差异。
- 哪些假设最敏感。
- 哪些证据会推翻预测。
```

## 8. 估值分析标准

必须同时做静态、动态和横向比较。

最低结构：

```text
5.1 静态估值：PE / PB / PS / EV 等可得指标。
5.2 动态估值：基于 2026E-2028E。
5.3 同业对比：可比公司、估值区间、溢价/折价原因。
5.4 场景估值：bear / base / bull。
5.5 估值结论：偏低 / 合理 / 偏高 / 已透支，以及为什么。
```

## 9. 技术分析标准

技术分析只作为市场状态观察，不替代基本面判断。

最低字段：

```text
- 数据日期。
- 当前价格。
- MA5 / MA10 / MA20 / MA60 / MA120 或可用趋势指标。
- 成交额、换手率、波动率。
- 关键支撑与阻力。
- 趋势状态：上升 / 震荡 / 下行 / 极端过热。
```

## 10. 情绪分析标准

分三层：

```text
宏观情绪：利率、汇率、大类资产、风险偏好。
行业/主题情绪：板块涨跌、资金热度、主题催化、产业新闻。
公司情绪：成交、换手、资金流、机构持仓、舆论分歧、公告反应。
```

情绪结论必须有日期，且只能作为观察，不可替代业绩证据。

## 11. 事件驱动标准

必须形成事件日历。

```markdown
| 日期/窗口 | 事件 | 影响变量 | 预期 | 超预期条件 | 低于预期风险 | 证据 |
|---|---|---|---|---|---|---|
```

## 12. 研究结论标准

结论必须分层：

```text
事实结论：已由证据支持。
推断结论：由事实和模型推出。
关键假设：未来需要验证。
风险和反证：可能推翻判断。
跟踪指标：后续更新需要看的数据。
```

替代样例中的“买入/推荐/观望”表达为：

```text
研究评级：high_conviction_watch / watch / neutral_watch / avoid_until_evidence
原因：...
```

## 13. Evidence Map 标准

报告末尾必须包含：

```markdown
| 结论 | claim_id / metric_id | evidence_id | 来源 | 日期 | 页码/表格 | 置信度 |
|---|---|---|---|---|---|---|
```

没有 evidence map 的报告不能进入 R3。
