# 项目日志总索引

Project: `03_Investment_System`
Last updated: 2026-07-01
Purpose: lightweight catalog for choosing project logs and guidance without reading their bodies.

## 默认规则

- 没有用户明确指令时，默认不读取任何项目日志或项目指引。
- 没有用户明确指令时，默认不创建、更新或追加任何项目日志或项目指引。
- 需要查找可选日志或指引时，只读取本索引；不要为了找文件而读取正文。
- 只有用户明确说 `读日志 <中文日志名>` 或 `读指引 <中文指引名>` 时，才读取被选中的那一个日志或指引。
- 只有用户明确说 `写日志`、`写交接`、`写决策`、`写指引`，或要求归档/记录时，才写入项目日志系统。
- 一个对话默认只维护一个主日志；若已存在本对话主日志，追加到同一文件。
- 当前有效的项目规则、项目构建计划、系统说明、skill/工作流指引放在 `guidance/`。

## 指令到动作

| 用户指令 | 允许读取 | 允许写入 |
|---|---|---|
| 未提日志/指引 | 不读取日志、指引或本索引 | 不写日志或指引 |
| `不读日志` / `不读指引` | 不读取日志、指引或本索引 | 仍需另行明确是否写 |
| `不写日志` / `不写指引` | 仅按用户明确选择读取 | 不写日志或指引 |
| `只列日志` | 只读取本索引 | 不写日志 |
| `只列指引` / `读指引菜单` | 只读取本索引 | 不写指引 |
| `读日志菜单` | 只读取本索引 | 不写日志 |
| `读日志 <中文日志名>` | 先用本索引定位，再只读被选中的一项 | 不写日志，除非另有 `写日志` |
| `读指引 <中文指引名>` | 先用本索引定位，再只读被选中的一项 | 不写指引，除非另有 `写指引` |
| `写日志` | 按任务需要读取同一主日志 | 写入一个主会话日志 |
| `写交接` | 按任务需要读取同一主日志 | 写入交接记录或追加主日志 |
| `写协作日志` | 按任务需要读取同一主日志 | 在 `handoffs/` 下写入给网页版 GPT/ChatGPT 的交接型协作日志 |
| `写决策` | 按任务需要读取同一主日志 | 写入决策记录或追加主日志 |
| `写指引` / `写项目规则` / `写项目计划` | 按任务需要读取相关指引 | 在 `guidance/` 下写入或更新当前指引 |

## 维护规则

- 新增、删除或合并日志/指引后，优先更新本索引。
- 本索引只写摘要和路径，不复制日志或指引正文。
- 若用户只想选择日志或指引，回答本索引中的候选项即可，不要主动展开正文。
- 若本索引或指引与 live 配置冲突，以 live 配置和验证命令为准。
