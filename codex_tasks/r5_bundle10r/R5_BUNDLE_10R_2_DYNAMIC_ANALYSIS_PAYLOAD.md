# 10R.2 — Dynamic analysis payload

## Goal

Replace company-specific prose assembly with a generic, structured analysis payload.

## Required analysis unit

Each required section must contain:

1. judgment;
2. at least two material facts;
3. causal mechanism;
4. economic implications;
5. counterevidence;
6. uncertainty boundary;
7. at least two watchpoints, including a quantified trigger;
8. display references.

## Acceptance

- Payload validation checks meaning-bearing fields, not only field presence.
- Forecast, valuation, risk, and conclusion sections cannot be omitted or compensated by other sections.
- The same code accepts a synthetic issuer without source-code edits.
