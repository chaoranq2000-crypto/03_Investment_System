# 14R-7 — Close or targeted backflow

## Close criteria

- all four contracts pass;
- all four use the same core runtime;
- truthfulness and determinism pass for all four;
- every case has an explicit qualification outcome;
- at least three exact-hash reviews mark the result `research_useful`;
- remaining limitations have owners, stages, evidence requests, and triggers;
- Bundle 13R state is unchanged unless new reviewed issuer evidence independently triggers its workflow;
- sample-quality and P2 remain controlled by the canonical release process.

## Failure behavior

Generate targeted backflow to evidence, analysis, forecast/valuation, report, or quality review. Never close with “write more”, “add more detail”, or an unowned TODO.
