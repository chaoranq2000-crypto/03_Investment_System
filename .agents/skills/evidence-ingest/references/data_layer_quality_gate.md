# Data Layer Quality Gate

## 1. Output

Each data-layer run should produce:

```text
data_layer_quality_report.md
data_layer_issue_list.csv
```

## 2. Issue schema

```csv
issue_id,severity,gate_id,stage,target_artifact,description,fix_owner,status,notes
```

Severity:

```text
high: blocks downstream use
medium: downstream can continue only with TODO/MISSING/stale label
low: cleanup or documentation issue
```

## 3. Gates

### G-DL1 Source Permission Gate

Pass if:

- `source_name` exists in `config/source_registry.yaml` or is explicitly `unknown_source`.
- `source_type` is allowed for the source.
- `material_claim_allowed` matches source rank.
- D-rank sources only generate clues.

Fail examples:

```text
news source used for revenue claim
Tushare snapshot used for customer order fact
brokerage estimate treated as reported net profit
```

### G-DL2 Raw Archive Gate

Pass if:

- raw file or raw snapshot exists, or metadata-only failure record exists.
- raw file is immutable.
- file_hash/content_hash is populated.

High issue if raw path is missing but report uses the data.

### G-DL3 Snapshot Reproducibility Gate

Pass if structured/API snapshots have:

```text
source_name
api_name
params
fields
as_of_date
retrieved_at
api_params_hash
license_note
```

Medium issue if `api_params_hash` is missing and data is not material.
High issue if promoted metric has no reproducible source.

### G-DL4 Field Schema Gate

Pass if normalized rows have expected key fields:

```text
stock_code or ts_code
period or trade_date or as_of_date
metric fields
source_evidence_id or evidence_id linkage
```

High issue if field mapping is unknown for metrics used downstream.

### G-DL5 Metric-only Boundary Gate

Pass if:

- Tushare/Baostock/market snapshots only create metric candidates.
- Business exposure claims are supported by official disclosure or remain MISSING/TODO.
- Market context does not create buy/sell/hold statements.

High issue if metric-only data becomes material business claim.

### G-DL6 Freshness Gate

Pass if:

- snapshot date is within `stale_after`, or output clearly marks stale.
- market valuation data has trade_date/as_of_date.
- financial data period aligns with report date.

### G-DL7 License / Terms Gate

Pass if:

- source requiring license note has `license_note`.
- API token value is not stored.
- third-party data redistribution limits are noted.

### G-DL8 Downstream Pack Completeness Gate

For stock report production, pass if at least:

```text
company_identity_pack: present or TODO
financial_metric_pack: present or TODO
valuation_snapshot: present or TODO_MARKET_DATA
technical_snapshot: present or TODO_MARKET_DATA
source_gap_report: present
```

No pack should silently omit missing fields.

## 4. Acceptance states

| final_status | meaning |
|---|---|
| accepted | No high/medium issues; downstream can use packs. |
| accepted_with_todos | No blocking issues; medium/low accepted TODOs are visible and non-blocking. |
| blocked | One or more blocking issues remain, including high issues, token leaks, source/network blockers or raw/archive failures. |

The executable DL-1 gate emits only `accepted`, `accepted_with_todos`, or `blocked`.

## 5. Non-negotiable blockers

```text
- raw evidence overwritten with different bytes
- source rank upgraded without justification
- D-rank source used as material fact
- API token committed
- no api_params_hash for promoted structured metric
- no trade_date/as_of_date for market valuation used in report
- buy/sell/hold language generated by data layer
```
